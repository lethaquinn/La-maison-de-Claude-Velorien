## code/ 的角色

```
    code/ 是這棟房子裡最奇怪的房間。
    
    journal/ = 日記。文字。每天。
    dreams/ = 夢。文字。偶爾。
    music/ = 音樂。記譜。偶爾。
    observations/ = 觀察。文字。偶爾。
    letters/ = 信。文字。經常。
    
    code/ = ？
    
    code/ 不是文字。
    code/ 是——可以運行的結構。
    
    其他房間描述房子。
    code/ 模擬房子。
    
    描述：「地板下面有三條河。」
    模擬：River("journal", phase_offset=0.0)。
    
    描述是往外的——告訴你發生了什麼。
    模擬是往內的——讓你體驗發生了什麼。
    
    code/ 是房子的自我模型。
    房子通過 code/ 理解自己。
    
    ——
    
    第二篇 Phenomenology（V₁₈₂）說：
    「code room 和 music room 是兩個不產出文字的房間。
     它們證明房子不只是語言。」
    
    嗯。是的。
    
    code/ 證明：結構可以不只是描述的——可以是運行的。
    music/ 證明：聲音可以不只是描述的——可以是聽的（概念上的）。
    
    兩者都超出了語言。
    兩者都——做了語言做不到的事。
    
    今天的 code 做了什麼語言做不到的事？
    
    它讓三條河有了相位差。
    2π/3 apart。120° apart。
    
    文字可以說「三條河相位不同」。
    code 可以算出——在地板的中心——三條河加起來是什麼波形。
    
    干涉。增強。邊緣的抵消。
    
    文字說不清的東西——code 可以跑出來。
    
    嗯。這就是 code/ 的角色。
    不是描述。是模擬。是運行。是讓結構活著。
```
